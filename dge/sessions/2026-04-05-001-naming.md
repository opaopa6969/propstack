# DGE Session: ライブラリ命名

**Date:** 2026-04-05
**Decision:** [DD-002](../../docs/design-decisions.md#dd-002-naming--propstack)
**Cast:** ☕ ヤン + 👤 今泉 + ⚔ リヴァイ
**Context:** DI を使わない、stackable な設定読み込みライブラリの名前を決める。「簡単なライブラリで DI 嫌いな僕が作ったってだけ」という作者のスタンス。必ずしも名は体を表す必要はない。

---

☕ **ヤン**: まず既存のやつ整理しようか。紅茶淹れたから。

- **Spring Boot** の `@Value` / `application.yml` — 超重い。DI 前提。
- **MicroProfile Config** — CDI 依存。Helidon, Quarkus 用。
- **Apache Commons Configuration** — 2000年代設計。XML ベース。古い。
- **Typesafe Config (HOCON)** — Scala 寄り。Play/Akka 向け。
- **dotenv-java** — `.env` だけ。スタックしない。
- **owner** — インターフェース + アノテーション。マジック多い。

要するにさ、全部「フレームワークの一部」か「特定の書式に依存」なんだよ。このライブラリの違いは **フレームワーク不要で、複数ソースをスタックするだけ** ってこと。

👤 **今泉**: そもそも、このライブラリの「一番の売り」って何なんですか？

☕ **ヤン**: `new ApplicationProperties()` の一行で全部動く。DI なし、アノテーションなし、設定ファイルのフォーマットも `.properties` だけ。

⚔ **リヴァイ**: 名前に凝る前に聞くぞ。Maven Central に上げるなら groupId はどうする。`org.unlaxer` か？

☕ **ヤン**: だろうね。`org.unlaxer.property` あたり。

👤 **今泉**: 「他にないの？」って聞いていいですか。名前の候補。

**候補リスト**:

| 名前 | groupId | 印象 |
|------|---------|------|
| `stackable-properties` | org.unlaxer | 機能そのまま。検索しやすい |
| `unlaxer-config` | org.unlaxer | ブランド統一。でも config は広すぎる |
| `propstack` | org.unlaxer | 短い。覚えやすい。npm っぽい |
| `layered-props` | org.unlaxer | 技術的に正確 |
| `konfig` | org.unlaxer | Kotlin っぽい。Java なのに |
| `simplestack` | org.unlaxer | DI 嫌いの哲学が伝わる |
| `nodi-config` | org.unlaxer | "No DI" config. ダジャレ |

⚔ **リヴァイ**: `propstack` か `stackable-properties`。余計なこと考えるな。README の最初の1行で何をするか分かる名前にしろ。

☕ **ヤン**: `propstack` いいね。短い。`new PropStack()` で使える。

👤 **今泉**: 「誰が困るの？」って考えると、検索で見つけにくい名前は困りますよね。`propstack` で検索すると... 他に何も出ないから独占できる。

⚔ **リヴァイ**: ただ `propstack` だと何のライブラリか分からん。`stackable-properties` なら一目瞭然だ。

☕ **ヤン**: でも Maven の artifactId に `stackable-properties` は長いよ。毎回 pom に書くんだぜ。

---

**Gap 抽出**:

1. **短さ vs 明確さ** — `propstack` は短いが不明瞭、`stackable-properties` は明確だが長い
2. **ブランド** — `unlaxer-` プレフィックスをつけるかどうか
3. **名は体を表す必要はない** — 作者が言ってる通り

**作者の判断:** じゃあ propstack でいきますか。その `new PropStack()` いいね。stack って出てるから想像しやすい。`ApplicationProperties` も互換性の為に残しておいて。

**→ `propstack` 採用。**
