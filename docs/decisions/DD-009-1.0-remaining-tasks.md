# DD-009: 1.0 Remaining Tasks

**Status:** Open  
**Date:** 2026-04-19  

---

## Context

PropStack 0.9.1 is published to Maven Central. The API is stable and the library is usable in production. However, certain quality gates should be in place before a 1.0 release.

---

## Open Items

### CI / Build

- [ ] **GitHub Actions workflow** — `mvn verify` on every push and PR. Currently there is no automated build. Any contributor can break the build without knowing it.
- [ ] **CI badge in README** — `[![Build](https://github.com/opaopa6969/propstack/actions/workflows/ci.yml/badge.svg)](...)`. Not added yet because the workflow does not exist.

### Documentation

- [ ] **Javadoc site** — publish to GitHub Pages. The `maven-javadoc-plugin` is already configured in `pom.xml`; deployment is not.
- [ ] **`PropStack.defaultSources()` stable API note** — currently underdocumented. Should appear in the main docs as a first-class feature.

### API

- [ ] **Evaluate: `trace()` show-all option** — current `trace()` stops at the first MATCH. A `traceAll(key)` variant that shows every source (for debugging layered configs) may be useful.
- [ ] **Evaluate: anonymous key catalog** — currently `validate()` only works with `KeyHolder` enums. A `KeyCatalog` class for registering `TypedKey` instances without enums would help for dynamic config schemas.
- [ ] **`TypedKey.longKey()` / `doubleKey()` factory naming** — the asymmetry with `string()`, `integer()`, `bool()` is a wart. Consider `long()` / `double()` via a static import alias (conflicts with Java keywords — likely keep as-is).

### Compatibility

- [ ] **Java 17 compatibility evaluation** — pom.xml targets Java 21. Records and sealed classes are used. Backport to 17 is possible but not prioritized.

---

## Not in Scope for 1.0

The following are explicitly out of scope:

| Feature | Reason |
|---------|--------|
| YAML support | Adds SnakeYAML dependency; violates zero-dependency goal |
| Hot reload | Bug-prone; restart is simpler and safer |
| Spring Boot auto-configuration | Dependency on Spring; defeats zero-dependency goal |
| IDE metadata JSON (`spring-configuration-metadata.json`) | Over-engineering for this scope |
| Nested config objects | Dot notation (`db.host`) already works with flat properties |

---

## Definition of Done for 1.0

- [ ] GitHub Actions CI passing on push/PR
- [ ] CI badge in README.md and README.ja.md
- [ ] All public API documented in Javadoc
- [ ] `defaultSources()` documented in getting-started.md and api-cookbook.md
- [ ] Zero open P0/P1 bugs
